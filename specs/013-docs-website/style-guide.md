# Dottie Style Guide

Dottie is a developer-friendly dotfile manager with a minimalist CLI aesthetic and approachable branding. This guide defines its visual identity and usage conventions.

---

## Logo & Badge

### Primary CLI Badge

- Format: `> dottie`
- Font: Monospaced, lowercase
- Colors:
    - Background: `#000000` (black)
    - Prompt: `#33cc66` (green)
    - Text: `#eaeaea` (off-white)
- Shape: Rounded rectangle
- Usage: GitHub README, documentation headers, CLI screenshots

### Status Badge

- Label: `dotfile manager`
- Style: Minimalist black-and-white
- Font: Sans-serif, clean and modern
- Usage: Project pages, tool listings, integrations

---

## Icon

### Mascot/Icon Mark

- Style: Minimalist black-and-white line art
- Shape: Circular or dot-based
- Personality: Neutral, friendly, tech-inspired
- Optional elements: subtle gear, terminal prompt, or dot grid
- Usage: App icons, favicons, splash screens

---

## Typography

- **CLI Text**: Monospaced, lowercase (`> dottie`)
- **Headings**: Sans-serif, bold
- **Body**: Sans-serif, regular weight
- Preferred fonts:
    - `JetBrains Mono`, `Fira Code`, or `Courier New` for CLI
    - `Inter`, `Roboto`, or `Helvetica Neue` for UI and docs

## Usage Guidelines

- Always lowercase `dottie` unless starting a sentence.
- Use the CLI badge (`> dottie`) to reinforce terminal-native identity.
- Keep icons and badges clean, neutral, and developer-friendly.

---

## Voice & Tone

- Friendly, clear, and pragmatic
- Speaks like a helpful CLI companion
- Avoids jargon unless context demands it
- Emphasizes reproducibility, composability, and transparency

---

## Examples

```bash
> dottie apply
> dottie status
> dottie install --profile dev
```
